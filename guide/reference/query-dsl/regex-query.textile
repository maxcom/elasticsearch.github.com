p. Matches documents that have fields matching a regular expression (*not analyzed*). Supported regular expressions are standard Java-platform ones. Note this query is very slow, as it needs to iterate over many terms. It is recommended to use this query only with Percolator or on small indices. The regex query maps to Lucene @RegexQuery@.

<pre class="prettyprint lang-js">
{
    "regex" : { "user" : "kimch[a-z]" }
}
</pre>

p. A boost can also be associated with the query:

<pre class="prettyprint lang-js">
{
    "regex" : { "user" : { "value" : "ki[a-z]+y", "boost" : 2.0 } }
}    
</pre>

p. Or :

<pre class="prettyprint lang-js">
{
    "regex" : { "user" : { "regex" : "ki[a-z]+y", "boost" : 2.0 } }
}    
</pre>

p. This multi term query allows to control how it gets rewritten using the "rewrite":multi-term-rewrite.html parameter.
